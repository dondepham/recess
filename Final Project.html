Final Project

Due date

Final project will be due on December 10th at 5pm.

How to submit

It should be in a subfolder in your public_html folder in a folder called final_project. In addition to this, your project will also be submitted to CCLE. Create a zip of all your files + a text file that contains a working link to your final project in the public folder. It is imperative that we have a link to your project so we can find it and grade it!!

Final vs. Final project

You may choose one of: a) Complete the final project or b) Take the final on the designated day. You may not choose both.

General requirements

Your project will be "web app" that demonstrates use of all major topics we learned during the quarter. Your grade may also improve if you demonstrate good use of minor topics.

list the of the topics/concepts that all final projects must demonstrate

All projects must demonstrate correct XHTML markup that validates, or HTML5.
  • Your page must use external CSS stylesheet and look pleasing.
  • Your project must incorporate cookies.
  • Your project must use JS.
  • Your project must use events.
  • Your project must use DOM methods in a significant way.
You must write a short (1 page max) instructions on how to use your online app, and what grade you are hoping to get for your project, and how you have satisified the requirements for that grade.

Requirements for an A grade

  • All the basic requirements satisfied
  • Must include an interaction with SQLite3 database using Ajax.
  • Your code must write something to the database using Ajax and then retrieve something from the database using Ajax. Timestamps should be used.
  • Page content should be refreshed via Ajax.

Requirements for B grade

  • All the basic requirements satisfied
  • For B grade your code does not need to use Ajax. Instead it can work much like our homework assignment #6. There could be a form to enter data.
  • This data is then written into a database and then your webpage will read info from the database and create a dynamic response page.

requirements for C grade

  • All the basic requirements satisfied
  • For C grade you need not use a database

What determines your grade?

Starting point for your grade is the grade whose requirements you satisfy. However, your grade may be slightly higher or lower, depending on some factors.

You will lose points

  • If there are errors or your work is sloppy or somehow just bare minimum, you may get an A-, B- etc. If the project just does not work. For example if you try to go for the A, but cannot get your Ajax to work, then you may end up with a C.

You may gain points

  • Your project is particularly impressive or creative
  • You incorporate some of the minor topics we discussed during the course

Minor topics

  • Your project will benefit if it uses some animation.
  • Your project will benefit if it uses XML in some way.
  • Additional benefit if your XML can be verified by a DTD.
  • Additional benefit if your project uses the session stuff.

Some examples

Forum

Build a simple forum that people can post notes on. Ajax version of this page is that people can simply click on a post button and a form is dynamically created that the user can fill then the post is stored in a database. Your Ajax should fetch new posts from the database every couple of seconds and append them to the forum. The forum should also always display how many seconds, minutes hours, days ago each post was posted. This info should be updated with Ajax every minute.

Interactive map

A map of some region that you can create pins in by clicking on locations. When a user clicks on a location a form pops up that user can write some info about that pin location. Subsequently when a user howevers over the pin location info about the location should pop up in a little info div above the pin.

Strawpoll

Create a version of http://www.strawpoll.me/

Be creative. I would rather see something different from these 3 examples. Creativity may boost a grade.
